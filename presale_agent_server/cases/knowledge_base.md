## 四、经典案例解析

### 案例1：CRM客户管理（高频场景）
**客户**：兄弟装饰（1001-5000人，建筑业）
**需求**：已用智能表格CRM模板，需改造+对接ERP数据
**方案**：智能表格调整+数据对接
**报价**：¥34,980

**关键字段**：
- 客户基本信息（名称、电话、行业等）
- 跟进记录（时间、内容、负责人）
- 合同信息（金额、期限、状态）
- 跟进阶段（线索→意向→成交→服务）

**智能表格搭建要点**：
1. 主表：客户台账
2. 子表：跟进记录、合同、订单
3. 关联：通过关联字段联动
4. 视图：按负责人筛选、按阶段分组

---

### 案例2：项目管理（高频场景）
**客户**：华鸿亿通（1-100人，家装/装饰）
**需求**：项目档案、里程碑留痕、概览图、员工日志关联
**方案**：纯智能表格搭建

**关键功能**：
1. 项目立项：客户信息、项目类型、负责人
2. 里程碑追踪：商务对接→设计→样板间→大货交付
3. 阶段视图：概览图显示各项目所处阶段
4. 员工日志：关联项目，自动汇总

---

### 案例3：财务/报销（中频场景）
**客户**：合骥基金（1-100人，金融）
**需求**：财务报销、发票查重、人事招聘审批

**关键功能**：

2. 发票查重：自动检测重复报销
3. 审批流：多级审批、条件分支

## 六、常见需求模式与响应话术

### 模式1：「想做一个XXX管理系统」
**响应方向**：
1. 先确认是内部管理还是对外服务
2. 问清楚核心流程（3-5步）
3. 问清楚使用人数和数据量
4. 给出参考案例

**话术**：
> "明白，您是想做一个[需求类型]。能简单说一下核心流程吗？比如从头到尾大概有几个步骤？"

### 模式2：「不知道能不能实现」
**响应方向**：
1. 先肯定大概率可以实现
2. 给1-2个类似案例
3. 约定演示时间

**话术**：
> "这个需求我们做过类似的案例，比如[案例名]，效果还不错。您方便的话我可以先给您演示一下，看看是不是您想要的。"

### 模式3：「要多少钱/怎么收费」
**响应方向**：
1. 说明定价因素（功能复杂度、数据量、定制程度）
2. 给出价格区间参考
3. 说明包含内容

**话术**：
> "智能表格定制的费用主要看功能复杂程度。简单的话几千元，复杂的可能要几万。我先了解一下您的具体需求，给您一个更准确的报价。"

### 模式4：「跟其他系统打通」
**响应方向**：
1. 确认是对接还是替换
2. 说明对接方式（API/同步表格/导出导入）
3. 评估对接难度

**话术**：
> "您说的打通是指想把现有系统数据和智能表格同步，还是想替换成智能表格？对接的话需要看对方系统是否开放接口。"

## 七、技术选型建议

| 需求复杂度 | 推荐方案 | 说明 |
|-----------|---------|------|
| **简单**（1-3张表，基础功能） | 纯智能表格搭建 | ¥2,000 ~ ¥6,000 |
| **中等**（多表关联，自动化） | 智能表格+API开发 | ¥6,000 ~ ¥20,000 |
| **复杂**（全流程，多系统对接） | 应用定制+智能表格 | ¥20,000 ~ ¥50,000 |
| **高度定制**（特殊逻辑，与ERP对接） | 纯定制开发 | ¥50,000+ |

**智能表格适合的场景**：
- 流程相对简单（5步以内）
- 数据量适中（<10万行）
- 需要多人协作
- 移动端访问为主
- 预算有限

**需要定制开发的场景**：
- 复杂审批流
- 需要对接多个外部系统
- 实时数据同步要求高
- 有特殊安全合规要求
