# 企业微信定制开发需求助手 - 服务端部署指南

## 功能概述

基于企业微信智能机器人长连接（WebSocket）的售前需求洞察 Agent，用户在企微里 @机器人 或单聊，即可完成从行业定向到细节规格的3轮对话，最终输出结构化需求洞察报告。

## 项目结构

```
presale_agent_server/
├── main.py               # 入口：WebSocket 客户端 + 消息路由
├── requirements.txt      # 依赖
├── README.md             # 本文档
├── agent/
│   ├── __init__.py
│   ├── dialogue.py       # 需求洞察 Agent 核心对话逻辑
│   └── state.py          # 对话状态机 + 会话管理
└── cases/
    ├── __init__.py
    └── library.py        # 案例知识库（简单关键词匹配 RAG）
```

## 快速开始

### 1. 安装依赖

```bash
cd presale_agent_server
pip install -r requirements.txt
```

### 2. 在企业微信管理后台创建机器人

1. 登录 [企业微信管理后台](https://admin.work.weixin.qq.com)
2. 进入 **应用管理** → **智能机器人**
3. 创建机器人，填写名称和描述
4. 在「API模式」中，选择 **长连接** 方式
5. 复制 **BotID** 和 **Secret**

> ⚠️ BotID 和 Secret 是长连接专用凭证，请妥善保管。

### 3. 启动服务

```bash
python main.py --bot-id YOUR_BOT_ID --secret YOUR_SECRET
```

### 4. 测试

在企业微信里，找到该机器人，发一条消息（或直接 @它），即可开始对话。

## 对话流程

| 阶段 | 目标 | Agent 提问示例 |
|------|------|--------------|
| **第0轮** | 破冰定向 | 你们是什么行业？想解决什么问题？ |
| **第1轮** | 业务全景 | 流程是怎样的？最头疼什么？期望哪些模块？ |
| **第2轮** | 细节深挖 | 数据规模？关键节点？使用角色？数据来源？分析维度？颗粒度？ |
| **第3轮** | 报告输出 | 结构化需求洞察报告 Markdown |

## 配置说明

| 参数 | 说明 |
|------|------|
| `--bot-id` | 智能机器人的 BotID（必填） |
| `--secret` | 长连接专用 Secret（必填） |
| `--cases-dir` | 额外案例库路径（可选，默认使用内置案例） |

## 添加自定义案例

编辑 `cases/library.py` 中的 `CASE_LIBRARY` 列表，或在指定目录下放置 `.json` 文件：

```json
[
  {
    "customer": "某客户名",
    "industry": "行业",
    "project_type": "项目类型",
    "pain_points": "痛点",
    "modules": ["模块1", "模块2"],
    "highlight": "核心亮点",
    "effect": "量化效果",
    "tags": ["标签1", "标签2"]
  }
]
```

## 升级方向

- [ ] **接入大模型 API**：当前是规则引擎模拟，可接入 Claude/GPT 等实现更自然的对话
- [ ] **向量检索**：将案例库升级为 Embedding + 余弦相似度检索
- [ ] **会话持久化**：把需求报告自动写回企业微信智能表格
- [ ] **多机器人路由**：支持多个机器人共用一个 Agent 实例
- [ ] **监控告警**：接入企业微信告警，连接断开时通知运维

## 常见问题

**Q: 连接断开后会自动重连吗？**
A: 会。服务端实现了指数退避重连机制，最长等待2分钟。

**Q: 一个 Bot 能同时服务多少用户？**
A: 每个 Bot 同时只保持一个长连接，但可以服务多个用户（通过 session_id 区分）。

**Q: 支持群聊 @机器人吗？**
A: 支持，消息回调中 `chattype` 会标识是 `group` 还是单聊。
